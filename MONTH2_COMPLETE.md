# Month 2 Complete — AI Application Developer Portfolio

Across four weeks (Weeks 5–8), starting from zero programming background eight weeks ago, the portfolio evolved from a single-purpose market analyzer into a single, production-style application that does three things in one place. **Week 5** added a full document Q&A (RAG) system alongside the existing market analyzer: users can upload PDF or text files, which are chunked and embedded with sentence-transformers and stored in ChromaDB; they then ask questions in natural language and receive Claude-generated answers grounded only in retrieved passages, with source citations and a confidence signal. The same app still runs stock and crypto analysis, asset comparison, saved analysis history, and a watchlist—all backed by FastAPI and SQLAlchemy. **Week 6** turned that RAG system into a production-ready one: sentence-aware chunking (no mid-sentence cuts), named collection groups so multiple documents can live in one collection, and three search scopes—this document only, whole collection, or all documents—with token budgeting to stay within model context limits and keyword-overlap re-ranking so exact terms (e.g. clause numbers, tickers) are not missed when vector search alone would rank them lower. **Week 7** focused on trust and operations: a 0–100 confidence score derived from ChromaDB retrieval distances with a color-coded bar and short hint so users know when to double-check an answer; categorized, user-facing error messages (e.g. “Ticker not found — check the symbol”) with suggestions instead of raw exceptions; phased loading states during document Q&A (“Searching document…”, “Selecting best passages…”, “Generating answer with Claude…”); and Alembic database migrations so schema changes are applied safely without dropping tables. **Week 8** added full authentication and per-user data: a standalone login/register page with guest access, JWT access tokens and bcrypt password hashing, a “who am I?” endpoint and auth guard so the main app loads only when the token is valid, and strict user data isolation—analyses, documents, history, and stats are filtered by user so each person sees only their own content; the frontend sends the Bearer token on every protected request and handles 401s by clearing storage, showing “Your session has expired,” and reloading to the login screen, with a small personalization touch (e.g. “Calculating indicators for Drew…”) so the app clearly reflects who is signed in.

## Technical skills acquired in Month 2

- **RAG pipeline (ChromaDB, sentence-transformers, chunking)** — End-to-end retrieval-augmented generation: chunk documents, embed with sentence-transformers (or Voyage), store in ChromaDB, embed the query, retrieve top-k, and pass context to Claude for answer generation.
- **Multi-document collection management** — Named collection groups; multiple documents per collection; metadata (e.g. `source_file`) so queries can target one doc, one collection, or all collections.
- **Cross-document semantic search** — Query scope: “this document only,” “whole collection,” or “all documents”; single API design for scoped retrieval and answer generation.
- **Token budgeting and re-ranking** — Cap total context tokens sent to the model; log usage and cost; apply keyword-overlap re-ranking on top of vector results for domain-specific and exact-match queries.
- **Alembic database migrations** — Versioned migrations instead of drop-and-recreate; safe schema evolution across dev and production (e.g. new columns, tables, indexes).
- **JWT authentication and bcrypt password hashing** — Issue and validate JWTs (e.g. `sub` = user id); hash passwords with bcrypt on register; verify on login; Bearer token on protected routes.
- **User data isolation with foreign key filtering** — `user_id` on analyses and documents; all list/detail/stats queries filter by `current_user.id` so users only see their own data.
- **User-facing error message categorization** — Map exceptions to categories (NOT_FOUND, RATE_LIMIT, INVALID_INPUT, AUTH, etc.); return `user_message` and `suggestion`; derive HTTP status from category.
- **Phased loading states and confidence scoring UI** — Show explicit phases during long operations (e.g. document Q&A); display a 0–100 confidence score with color-coded bar and short hint so users know when to trust or verify the answer.
